# Cloud-Vault 前端实现方案

## 一、目标与范围

Cloud-Vault Web 端使用 Vue3 + TypeScript + Vite 实现，UI 组件指定使用 shadcn-vue，状态管理使用 Pinia。前端需要覆盖普通用户记账主流程和系统管理员的用户、邀请码、系统设置管理。

MVP 首屏不是营销页，而是登录后直接进入可操作的记账工作台：

- 首页 Dashboard：本月概览、快速记账、近期账单、预算进度。
- 账单：列表、筛选、创建、编辑、删除、恢复。
- 账户：账户余额、账户详情、账户维护。
- 统计：收支趋势、分类占比、排行。
- 设置：个人设置、分类管理、主题语言。
- 管理后台：邀请码、用户、系统设置。

## 四、关键配置文件

### 4.1 `components.json`

shadcn-vue 组件生成配置，建议使用别名：

```json
{
  "style": "new-york",
  "typescript": true,
  "tailwind": {
    "config": "tailwind.config.ts",
    "css": "src/assets/styles/globals.css",
    "baseColor": "neutral",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/utils",
    "ui": "@/components/ui",
    "lib": "@/utils"
  }
}
```

### 4.2 `src/utils/cn.ts`

用于合并 Tailwind class：

```ts
import { clsx, type ClassValue } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

### 4.3 `src/services/http.ts`

统一处理：

- `baseURL = import.meta.env.VITE_API_BASE_URL || '/api/v1'`
- 自动附加 `Authorization: Bearer <token>`
- 自动附加 `Accept-Language`
- 统一解析后端响应包裹
- `401` 清理登录状态并跳转登录页
- `403` 跳转无权限页或弹出提示
- 展示后端 `message`
- 支持 `Idempotency-Key`

### 4.4 `.env`

```text
VITE_API_BASE_URL=/api/v1
VITE_APP_NAME=Cloud-Vault
VITE_DEFAULT_LOCALE=zh-CN
```

## 五、状态管理设计

### 5.1 Pinia Store 划分

| Store | 职责 | 是否持久化 |
|-------|------|------------|
| `auth.store.ts` | token、当前用户、登录状态、系统角色 | 是 |
| `app.store.ts` | 全局加载态、侧栏展开、设备断点、全局错误 | 是 |
| `theme.store.ts` | 亮色/暗色/跟随系统、主题色 | 是 |
| `book.store.ts` | 当前账本、账本列表、账本角色 | 是 |
| `transaction-filter.store.ts` | 账单筛选条件、日期范围、搜索词 | 是 |

### 5.2 服务端状态

接口数据使用 `@tanstack/vue-query` 管理，Pinia 不直接保存所有服务端列表数据。

示例：

| 数据 | 缓存 key |
|------|----------|
| 当前用户 | `['me']` |
| 账本列表 | `['books']` |
| 账户列表 | `['accounts', bookId]` |
| 分类列表 | `['categories', bookId]` |
| 账单列表 | `['transactions', bookId, filters]` |
| 报表总览 | `['reports', 'summary', bookId, dateRange]` |

写操作成功后按 key 精准失效：

- 创建账单：失效 `transactions`、`accounts`、`reports`、`budgets`。
- 修改分类：失效 `categories`、`transactions`。
- 修改账户：失效 `accounts`、`transactions`。

## 六、多语言方案

### 6.1 支持语言

MVP 支持：

| 语言 | locale |
|------|--------|
| 简体中文 | `zh-CN` |
| English | `en-US` |

默认语言：

1. 优先读取用户设置。
2. 其次读取 localStorage。
3. 再读取浏览器语言。
4. 最后 fallback 到 `zh-CN`。

### 6.2 文案命名

统一使用点号路径：

```ts
export default {
  common: {
    save: '保存',
    cancel: '取消',
    delete: '删除'
  },
  auth: {
    login: '登录',
    register: '注册',
    inviteCode: '邀请码'
  },
  transaction: {
    expense: '支出',
    income: '收入',
    transfer: '转账'
  }
}
```

### 6.3 使用规范

- 组件内不写死中文。
- 错误码使用 `errors.INVALID_INVITE_CODE` 这类 key 映射。
- 金额、日期、数字格式通过 `Intl.NumberFormat` 和 `Intl.DateTimeFormat`。
- 路由标题从 i18n 读取。

## 七、主题与颜色风格

### 7.1 总体风格

Cloud-Vault 当前前端风格对齐 `E:\Pythonwork\mimoinvite`：黑白高对比、方角强边框、系统字体、等宽数字、轻量上移动效。它仍然是财务工具，界面应安静、清晰、密度适中，避免营销页式大 Hero、过度装饰、柔和圆角卡片和强烈渐变。设计重点：

- 信息优先，适合长期使用和快速扫描。
- 数据卡片克制，更像表格分区和边框网格，避免卡片套卡片。
- 图表色彩稳定，收入、支出、转账语义明确。
- 移动端优先，桌面端增强信息密度。
- 组件默认方角，主操作使用黑白反色；不要回到蓝色圆角 shadcn 默认观感。
- 标题使用粗体系统字体，指标金额使用等宽字体，标签/kicker 使用小号等宽大写。

### 7.2 颜色策略

基础使用 mimoinvite 风格的 `--app-*` CSS Variables，再映射到 Tailwind/shadcn 变量。整体以黑、白、灰为主，语义色只用于收入、支出、转账、预算状态，不作为大面积装饰色：

| 用途 | 亮色 | 暗色 | 说明 |
|------|------|------|------|
| 背景 | `#FFFFFF` | `#050505` | 页面底色 |
| 面板 | `#FFFFFF` | `#0B0B0B` | 表单、分区、弹窗 |
| 正文 | `#111111` | `#F5F5F5` | 主文本和强边框 |
| 次级文本 | `#666666` | `#A3A3A3` | muted |
| 弱文本 | `#9A9A9A` | `#737373` | faint |
| 主边框 | `#111111` | `#F5F5F5` | 强分割线、按钮边框 |
| 弱边框 | `#D7D7D7` | `#303030` | 表格行线、次级分割 |
| 次级底色 | `#F5F5F5` | `#121212` | hover、空状态、提示 |
| 强次级底色 | `#EEEEEE` | `#1C1C1C` | hover 强调 |
| 反色底 | `#111111` | `#F5F5F5` | 主按钮、选中导航 |
| 反色文本 | `#FFFFFF` | `#050505` | 主按钮文字 |
| 收入 | `#15803D` | `#22C55E` | income |
| 支出 | `#B91C1C` | `#EF4444` | expense |
| 转账 | `#1D4ED8` | `#3B82F6` | transfer |
| 预算警告 | `#B45309` | `#F59E0B` | warning |

不要让界面被单一蓝色或紫色主导。蓝色不能作为主视觉，只能作为转账或少量状态色。按钮、导航选中态、表格强调优先使用黑白反色。

- 选项浮层优先使用图标 + 名称 + 必要副信息的高密度行布局；在右侧快速记账区域，账户/属性选择从底部向上展开，避免遮挡金额键盘。

模式：

|------|------|
| `light` | 固定亮色 |

## 八、动画与交互效果

### 8.1 动画原则

- 动画服务于状态变化，不做纯装饰。
- 时长保持短：常规 120-200ms，页面切换 180-240ms。
- 使用 `transform` 和 `opacity`，避免频繁改 layout 属性。
- 尊重 `prefers-reduced-motion`，用户减少动态效果时禁用复杂动画。

### 8.2 动画类型

| 场景 | 效果 | 时长 |
|------|------|------|
| 页面切换 | 轻微 fade + translateY(4px) | 180ms |
| 顶部偏好切换 | 颜色短过渡 + 控制区滑扫反馈 | 180-420ms |
| Dialog/Sheet 打开 | fade + scale(0.98 -> 1) | 160ms |
| 列表项进入 | stagger fade-in，间隔 20ms | 160ms |
| 按钮点击 | scale(0.98) + 快速恢复 | 100ms |
| 筛选条件变化 | 列表 skeleton -> 内容淡入 | 160ms |
| Toast | slide-in + fade | 180ms |
| 预算进度 | width 从旧值过渡到新值 | 240ms |
| 数字变化 | 金额 counter 过渡 | 300ms |
| 快速记账成功 | 输入框清空 + 成功 toast，可撤销 | 200ms |

### 8.3 动画实现方式

优先级：

1. Tailwind transition class。
2. Vue `<Transition>` / `<TransitionGroup>`。
3. ECharts 内置动画。
4. 复杂页面级动效再考虑 `@vueuse/motion`。

基础 class：

```css
.page-enter-active,
.page-leave-active {
  transition: opacity 180ms ease, transform 180ms ease;
}

.page-enter-from,
.page-leave-to {
  opacity: 0;
  transform: translateY(4px);
}

@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 1ms !important;
    transition-duration: 1ms !important;
    scroll-behavior: auto !important;
  }
}
```

## 九、页面与功能实现

### 9.1 登录/注册页

功能：

- 登录：邮箱、密码。
- 注册：邮箱、密码、确认密码、邀请码。
- 注册提示：邀请码由管理员分发。
- 支持暗亮模式和语言切换。
- 注册成功后回到登录页或直接登录，具体跟后端策略保持一致。

组件：

- `Card`、`Input`、`Button`、`Form`、`Alert`、`Select`。

需要接口：

- `GET /api/v1/public/config`
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`

### 9.2 主布局

普通用户应用入口固定为 `/`，使用 `UserLayout`。管理后台不挂在普通用户布局下，避免管理功能和日常记账导航混在一起。

桌面端：

- 左侧导航。
- 顶部栏右侧承载账本切换、亮色/暗色按钮和语言选择；快速记账在首页右侧工作台完成，不再依赖顶部按钮。
- 内容区按页面滚动。

移动端：

- 顶部栏 + 底部导航。
- 快速记账使用底部 Drawer。
- 筛选器使用 Sheet。

导航项：

- 首页
- 账单
- 统计
- 账户
- 预算
- 设置

管理员用户可以在用户菜单中看到「进入管理后台」，点击后跳转 `/admin`，但普通侧栏不直接承载管理子路由。

### 9.3 Dashboard

模块：

- 本月收入、支出、结余。
- 净资产概览。
- 快速记账/AI 输入。
- 最近账单。
- 预算进度。
- 分类支出 Top 5。

视觉：

- 概览卡片不超过 4 张。
- 图表和列表使用 full-width section，不做卡片套卡片。
- 金额使用等宽数字或 `font-variant-numeric: tabular-nums`。
- 宽屏 Dashboard 采用高密度记账工作台：左侧账本/资产，第二列按日期分组流水，第三列账单详情，右侧快速录入/分类/计算器。
- 首页不保留大面积空白内容区；未选中账单时仅显示紧凑空状态，优先让账户、流水、详情和录入口同时可见。
- 右侧快速录入区必须能直接创建账单：类型、金额、账户、分类/转入账户、日期、备注都在当前面板完成，首页不再依赖额外的「记一笔」跳转按钮。
- 空账本状态下，右侧快速录入区提供默认账户和基础分类初始化入口，避免用户进入账本后无法直接开始记录。

### 9.4 账单页

功能：

- 日期范围筛选。
- 类型筛选：全部、支出、收入、转账。
- 分类、账户、标签筛选。
- 关键字搜索。
- 按日期分组。
- 创建、编辑、删除、恢复。
- 批量选择预留。

组件：

- `Tabs`、`Popover`、`Calendar`、`Select`、`Command`、`Sheet`、`Dialog`、`ScrollArea`。

交互：

- 桌面端编辑使用 Dialog。
- 移动端编辑使用 Drawer。
- 删除后 Toast 提供 30 秒恢复入口。

### 9.5 快速记账表单

字段：

- 类型：支出/收入/转账。
- 金额。
- 账户。
- 转入账户，转账时显示。
- 分类，支出/收入时显示。
- 日期。
- 备注。
- 标签。

规则：

- 金额必须为整数 minor unit。
- 支出和收入必须选择分类。
- 转账必须选择转入账户，且不能与转出账户相同。
- 账户、分类、标签只显示当前账本下的数据。

### 9.6 账户页

功能：

- 账户列表。
- 余额展示。
- 账户详情账单流水。
- 新增、编辑、归档账户。
- 信用卡账单日和还款日字段预留。

视觉：

- 账户列表使用图标 + 名称 + 类型 + 余额。
- 收入/支出变化不在账户卡里做复杂装饰，避免干扰余额扫描。

### 9.7 统计页

图表：

- 收支趋势折线图。
- 分类支出环图。
- 分类/商户排行。
- 日历热力数据，P1。
- 净资产趋势，P1。

ECharts 主题：

- 根据 `theme.store.ts` 切换亮色/暗色。
- 图表颜色使用语义色数组，不使用随机色。
- 图例、坐标轴、tooltip 文案走 i18n。

### 9.8 设置页

模块：

- 个人信息。
- 偏好设置：语言、主题、默认货币。
- 分类管理。
- 数据管理：导出、备份预留。
- 关于。

### 9.9 管理后台

管理后台入口固定为 `/admin`，使用独立的 `AdminLayout`，管理员可见。管理后台不依赖当前账本，也不展示普通记账侧栏。

页面：

- 管理概览，可选，MVP 可直接重定向到 `/admin/users`。
- 用户管理。
- 邀请码管理。
- 系统设置。

邀请码管理：

- 创建邀请码。
- 明文只在创建成功弹窗显示一次。
- 列表不显示明文，只显示状态、使用次数、有效期、限定邮箱、备注。
- 查看使用记录。
- 禁用或删除未使用邀请码。

组件：

- `Table`、`Dialog`、`Badge`、`DropdownMenu`、`Alert`、`Tooltip`。

## 十一、路由设计

```text
/login
/register
/
/transactions
/accounts
/reports
/budgets
/settings
/admin
/admin/users
/admin/invites
/admin/settings
/403
/404
```

路由分组：

| 分组 | 入口 | Layout | 说明 |
|------|------|--------|------|
| 公开路由 | `/login`、`/register` | `AuthLayout` | 登录、注册、公开配置 |
| 普通用户路由 | `/` | `UserLayout` | 日常记账应用，首页就是 Dashboard |
| 管理后台路由 | `/admin` | `AdminLayout` | 用户、邀请码、系统设置管理 |

普通用户路由：

```text
/                 -> Dashboard
/transactions     -> 账单
/accounts         -> 账户
/reports          -> 统计
/budgets          -> 预算
/settings         -> 设置
```

管理后台路由：

```text
/admin            -> 重定向 /admin/users 或管理概览
/admin/users      -> 用户管理
/admin/invites    -> 邀请码管理
/admin/settings   -> 系统设置
```

路由守卫：

1. 未登录访问私有页，跳转 `/login`。
2. 已登录访问 `/login`，跳转 `/`。
3. 访问 `/admin` 或 `/admin/*` 必须登录且 `systemRole = admin`，否则跳转 `/403`。
4. 普通用户路由需要当前账本；没有当前账本时，先加载账本列表，仍为空则创建默认账本或提示初始化异常。
5. 管理后台路由不要求当前账本，避免管理员进入 `/admin` 时被账本初始化流程阻塞。

## 十二、响应式设计

断点：

| 断点 | 宽度 | 布局 |
|------|------|------|
| mobile | `< 768px` | 底部导航、Drawer 表单 |
| tablet | `768px - 1023px` | 折叠侧栏、两列局部布局 |
| desktop | `>= 1024px` | 左侧导航、主内容区，重复信息优先三到四栏 |
| wide | `>= 1440px` | 四栏网格为默认密度，避免只用两栏造成信息过稀 |

要求：

- 文本不能溢出按钮、卡片和表格单元格。
- 表格在移动端改为列表卡片或横向 ScrollArea。
- 固定格式控件要有稳定尺寸，避免 hover 或加载态造成布局跳动。
- 账单列表使用虚拟滚动预留，MVP 可先普通分页。
- Web 宽屏下，账户、指标、近期账单卡片等重复内容默认使用四栏网格；只有详情页、长表单和图表主画布才允许占用两栏或整行。
- Web 端首页不能再使用松散两栏大面板；默认应参考账本软件的四区信息架构，减少内边距、压缩导航宽度，并让快速录入始终在首屏可达。
